## Overview

Package 3 completes the Publishing Engine with a deterministic mock processing layer that simulates publishing to LinkedIn, X, Instagram, and Facebook. It introduces durable queue state transitions, exponential retry with backoff, dead-letter handling, idempotent processing, workspace-scoped settings, and a full publishing audit trail.

---

## What's Included

### New Backend Services
- `PublishingQueueService` — Queue processing orchestration with state machine, retry budget, and dead-letter routing
- `MockPublishingProvider` — Deterministic mock with `success`, `transient_failure`, and `permanent_failure` modes
- Workspace-scoped processing settings API (owner/admin only)

### Queue State Machine
| State | Description |
|-------|-------------|
| `processing` | Active publishing attempt |
| `published` | Successfully published |
| `retry_pending` | Transient failure; waiting for retry window |
| `dead_letter` | Exhausted retry budget |
| `failed` | Permanent failure |
| `cancelled` | User-cancelled job |

### Key Features
- **Exponential Retry Backoff:** 4-attempt budget with 2^n minute delays (2, 4, 8, 16 min)
- **Dead-Letter Handling:** Jobs move to dead letter after exhausting retry budget
- **Idempotency:** Duplicate processing requests return existing state without side effects
- **Cancellation:** In-flight jobs can be cancelled at any state
- **Audit Trail:** Every state transition is recorded with timestamp, actor, and metadata
- **Role Authorization:** Only owners and admins can process, retry, or cancel jobs
- **Workspace Isolation:** Full data isolation across workspaces

### Frontend Updates
- Process Next button on Publishing Queue screen
- Per-job process, retry, and cancel controls
- New status badges: processing, retry pending, dead letter
- Provider attempt count and audit history display

### New API Endpoints
| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/v1/workspaces/{id}/publishing/process` | Process queue (Process Next) |
| POST | `/api/v1/workspaces/{id}/publishing/drafts/{id}/process` | Process specific draft |
| POST | `/api/v1/workspaces/{id}/publishing/drafts/{id}/retry` | Retry failed draft |
| POST | `/api/v1/workspaces/{id}/publishing/drafts/{id}/cancel` | Cancel draft |
| GET | `/api/v1/workspaces/{id}/publishing/drafts/{id}/audit` | Publishing audit history |
| GET | `/api/v1/workspaces/{id}/publishing/settings` | Get processing settings |
| PUT | `/api/v1/workspaces/{id}/publishing/settings` | Update processing settings |
