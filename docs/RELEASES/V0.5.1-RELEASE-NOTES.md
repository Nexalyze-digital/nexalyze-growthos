# GrowthOS v0.5.1 Release Notes

## Highlights

- Live local PostgreSQL 17 validation.
- `psycopg` PostgreSQL driver support.
- SQLAlchemy pool pre-ping and configurable pool settings.
- Alembic upgrade, downgrade, and re-upgrade validation.
- PostgreSQL backup and restore helper scripts.
- JSON migration validation against PostgreSQL-backed tests.
- Automatic frontend access-token refresh.
- Refresh request de-duplication and one-time retry behavior.
- Session clearing after refresh failure.
- Workspace selection preservation after refresh.
- Role-aware frontend controls for viewer/editor/admin/owner behavior.
- CI-ready Playwright smoke workflow.

## Validation

- Backend tests: 50 passed.
- PostgreSQL migration: passed locally.
- PostgreSQL backend suite: passed locally.
- Backup/restore rehearsal: passed locally.
- Frontend lint: passed.
- Frontend build: passed.
- Playwright smoke: passed locally.

## Known Limitations

- Local PostgreSQL validation does not prove hosted production readiness.
- No MFA, email verification, password reset, invite flow, or admin portal yet.
- Tokens remain in localStorage.
- Rate limiting remains in-memory.
- CI workflow requires dependency installation from public registries.
- `npm audit` reports a moderate PostCSS advisory through Next; remediation should use a compatible Next upgrade path rather than the breaking automatic downgrade.
