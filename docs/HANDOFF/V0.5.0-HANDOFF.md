# GrowthOS v0.5.0 Handoff

## Release

GrowthOS v0.5.0 - Platform Identity, Workspaces, And Database

## Branch

`feature/platform-identity-database`

## Features Implemented

- User registration.
- Login and logout.
- JWT-style access tokens.
- Refresh tokens with rotation and revocation.
- Password hashing and validation.
- Current-user endpoint.
- Organizations and workspaces.
- Workspace memberships and roles.
- Workspace switcher.
- Protected frontend dashboard.
- Workspace-scoped Brand Brain.
- Workspace-scoped Research Hub.
- Workspace-scoped Brand Brain context for Content Studio.
- SQLAlchemy database layer.
- Alembic migration scaffolding.
- JSON-to-database migration utility.
- Audit events.
- Security headers.
- CORS header/method restriction.

## Startup Commands

Backend:

```powershell
cd apps\api
.\.venv\Scripts\python.exe -m uvicorn main:app --host 127.0.0.1 --port 8000
```

Frontend:

```powershell
cd apps\web
npm run dev
```

Migration dry run:

```powershell
cd apps\api
.\.venv\Scripts\python.exe scripts\migrate_json_to_db.py --workspace-id <workspace-id> --dry-run
```

Migration live local test:

```powershell
cd apps\api
.\.venv\Scripts\python.exe scripts\migrate_json_to_db.py --workspace-id <workspace-id>
```

## Deployment Prerequisites

- Set `JWT_SECRET_KEY` outside source control.
- Set `DATABASE_URL`.
- Use PostgreSQL for production deployments after completing the PostgreSQL validation checklist in `docs/AI/PLATFORM-IDENTITY-DATABASE.md`.
- Install production database driver in the deployment environment.
- Configure CORS origins for the deployed frontend.
- Confirm secrets are not logged or exposed to the frontend.

## Rollback Considerations

- Revert the v0.5.0 merge commit.
- Restore JSON data from `.bak` files if a migration was performed.
- Preserve database backups before destructive environment operations.
- In production, rollback should be paired with database backup/restore procedures.
- Local development currently auto-creates tables at API startup for convenience. Run Alembic against a fresh database or stamp an existing local database before applying migrations manually.
