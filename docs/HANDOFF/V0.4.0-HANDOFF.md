## Known Limitations

- No live web crawling.
- No source fetching.
- No citation verification.
- No external search API integrations.
- JSON persistence is local and single-user only.
- No authentication.
- No workspace ownership.
- No database persistence.
- No vector search.
- Playwright smoke is local and not yet CI-enforced.

## Rollback Considerations

- The feature is isolated behind new Research Hub backend routes and frontend dashboard sections.
- JSON research data is stored under `apps/api/data` by default and is ignored by git.
- Rollback can be performed by reverting the v0.4.0 squash merge commit.
- If local research data causes confusion after rollback, remove `apps/api/data/research-runs.json`.
- Previous AI Content Studio and Brand Brain functionality should remain unaffected by rollback.

## Startup Commands

Backend:

```powershell
cd apps\api
.\.venv\Scripts\python.exe -m uvicorn main:app --host 127.0.0.1 --port 8000
```

Frontend:

```powershell
cd apps\web
npm run dev
```

Ollama validation:

```powershell
ollama --version
ollama list
```

Recommended local AI environment:

```powershell
$env:AI_PROVIDER = "ollama"
$env:AI_FALLBACK_PROVIDER = "mock"
$env:OLLAMA_BASE_URL = "http://localhost:11434"
$env:OLLAMA_MODEL = "llama3.2:latest"
```

## Deployment Prerequisites

- Python dependencies installed in `apps/api/.venv`.
- Frontend dependencies installed in `apps/web/node_modules`.
- Ollama installed for live local AI validation.
- At least one supported local model installed.
- GitHub PR reviewed and merged into `main`.
- GitHub Release `v0.4.0` created after merge.
- Roadmap updated after merge to mark Research Hub complete and Publishing Engine next.
